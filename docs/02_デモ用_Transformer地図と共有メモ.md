# デモ用 Transformer 地図（8ノード）＋ 他AI向け共有メモ

作成目的: **10分発表でアプリを実際に動かす**ための見本データ。学術的な完全性は不要だが、**説明の流れは破綻しない**程度に整える。

---

## 0. 設計方針（レビュー反映）

| 論点 | 採用する設計 | 理由 |
|------|-------------|------|
| Word2Vec の次 | **Seq2seq + Attention** を1ノードで挟む | Word2Vec の埋め込みを入力にした **RNN 系 seq2seq** が先にあり、そこへ Attention が載った方が歴史的に正確。Word2Vec → Transformer 直結はしない。 |
| LoRA の親 | **BERT と GPT-2 の両方** | LoRA は特定モデル専用ではなく大規模モデル全般の微調整手法。 |
| RAG の親 | **GPT-2 のみ（デモ上）** | 厳密には生成モデル一般の応用だが、地図を単純に保つ。口頭で「Transformer 系の生成モデル向け」と補足する。 |
| T5 | **Transformer から1枝のみ・下流なし** | デモで非関連化しやすい。枝先に何もつながない「葉」にする。 |

**MVP スコープ外（将来案）:** 「ノードをクリックすると詳しい説明を聞ける」— 現状は summary / メモを手入力。音声・AI解説は未実装。

## 1. 8つのノード（手法・論文）

| # | ノード名（グラフ上のタイトル） | 年 | 一言概要（summary にそのまま使える） |
|---|-------------------------------|-----|--------------------------------------|
| 1 | **Word2Vec** | 2013 | 単語をベクトルで表現。意味の近さを数値化した。 |
| 2 | **Seq2seq + Attention** | 2014–15 | RNN で系列変換（seq2seq）。翻訳で「どこを見るか」を学習する Attention を導入。 |
| 3 | **Transformer** | 2017 | RNN を使わず Attention のみで系列を処理。並列計算がしやすい。 |
| 4 | **BERT** | 2018 | 双方向の文脈で事前学習。理解・分類向き（エンコーダ系）。 |
| 5 | **GPT-2** | 2019 | 左から右へ予測する生成モデル（デコーダ系）。 |
| 6 | **T5** | 2020 | あらゆる NLP を「テキスト→テキスト」に統一（encoder-decoder）。 |
| 7 | **LoRA** | 2021 | 巨大モデルを低コストで微調整。BERT / GPT 系などに適用。 |
| 8 | **RAG** | 2020 | 検索した文書を参照しながら LLM が回答（生成モデルの応用）。 |

**デモ用「非関連」候補（9個目・任意）:** `ResNet` — 「研究は NLP、画像 CNN は今は追っていない」と説明。

## 2. 接続（エッジ）— 正確な流れのダイアグラム

### ASCII 図

```
Word2Vec
    │
    ▼
Seq2seq + Attention
    │
    ▼
Transformer ─────┬──────────────┐
    │            │              │
    ▼            ▼              ▼
  BERT         GPT-2            T5  （下流なし・非関連化向き）
    │            │
    │            ├──────────────┐
    ▼            ▼              ▼
  LoRA ◄─────────┘            RAG
    ▲
    └── GPT-2 からも入力（微調整の共有先）
```

※ LoRA は **BERT → LoRA** と **GPT-2 → LoRA** の2本。RAG には **GPT-2 → RAG** のみ（デモ簡略化）。

### エッジ一覧

| from | to | ラベル（任意） | 口頭で言う一言 |
|------|-----|----------------|----------------|
| Word2Vec | Seq2seq + Attention | prerequisite | 埋め込みを入力に、RNN で系列処理する時代へ。 |
| Seq2seq + Attention | Transformer | extends | Attention を中心に据え、RNN を排したのが Transformer。 |
| Transformer | BERT | branch | エンコーダ中心の事前学習。 |
| Transformer | GPT-2 | branch | デコーダ中心の生成。 |
| Transformer | T5 | branch | text-to-text 統一フレームワーク（今回は追わない、など）。 |
| BERT | LoRA | fine-tuning | 大規模モデルを軽くチューニング。 |
| GPT-2 | LoRA | fine-tuning | 生成モデル側も同様に LoRA で適応。 |
| GPT-2 | RAG | application | 生成モデルに検索を足す代表例（※厳密には生成モデル一般）。 |

**意図的に引かないエッジ**

| つながないペア | 理由 |
|----------------|------|
| T5 → 下流すべて | デモで葉ノード化・非関連トグル用。 |
| LoRA → RAG | 実務ではあり得るが、10分デモでは枝が増えすぎる。口頭で補足可。 |
| Word2Vec → Transformer | 中間の seq2seq 時代を飛ばすと説明が弱い。 |
