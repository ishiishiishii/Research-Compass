### `graph-utils.ts` — DB の形を React Flow の形に変換

React Flow が欲しいのは `position: { x, y }` です。DB は `position_x`, `position_y` です。

```13:31:frontend/src/lib/graph-utils.ts
export function toFlowNodes(
  nodes: PaperNode[],
  relevantOnly: boolean,
  options?: { linkSourceId?: string | null; showHandles?: boolean },
): PaperFlowNode[] {
  const { linkSourceId = null, showHandles = false } = options ?? {}
  return nodes
    .filter((n) => !relevantOnly || n.is_relevant)
    .map((n) => ({
      id: n.id,
      type: 'paper',
      position: { x: n.position_x, y: n.position_y },
      data: {
        node: n,
        isLinkSource: linkSourceId === n.id,
        showHandles,
      },
      style: { opacity: n.is_relevant ? 1 : 0.35 },
    }))
}
```

| 行 | 意味 |
|----|------|
| `filter` | Step 7 用。Step 1 では `relevantOnly` は常に false |
| `id: n.id` | React Flow がノードを識別する ID（DB の UUID と同じにする） |
| `type: 'paper'` | 後述の `nodeTypes={{ paper: PaperNodeComponent }}` と対応 |
| `position` | 画面上の座標。DB の `position_x/y` をここに載せ替える |
| `data.node` | カスタムノード内で論文情報を表示するための入れ物 |

```13:38:frontend/src/lib/api/nodes.ts
export async function createNode(
  userId: string,
  input: PaperNodeInput,
): Promise<PaperNode> {
  if (!supabase) throw new Error('Supabase が未設定です')
  const { data, error } = await supabase
    .from('nodes')
    .insert({
      user_id: userId,
      title: input.title,
      ...
      position_x: input.position_x ?? 0,
      position_y: input.position_y ?? 0,
    })
    .select()
    .single()
  if (error) throw error
  return normalizeNode(data)
}
```

| 行 | 意味 |

| `user_id: userId` | 誰のノードか |
| `.select().single()` | 追加された行をそのまま返してもらう |
| `normalizeNode` | DB の `prerequisites` などをフロント用に整える |

### ⑤ `dbNodes` と `nodes` の 2 段構造


```6:26:frontend/src/contexts/AuthContext.tsx
export function AuthProvider({ children }: { children: ReactNode }) {
  const [session, setSession] = useState<Session | null>(null)

    if (!supabase) return

      setLoading(false)

      data: { subscription },
```
